# Code Review Notes

Дата ревью: 2026-03-26

## Область проверки
- `cmd/client/main.go`
- `cmd/server/main.go`
- `pkg/turn_discovery/turn_discovery.go`
- `pkg/dpi_evasion/dpi_evasion.go`
- `pkg/probe/probe.go`
- `pkg/v2ray_bridge/v2ray_bridge.go`

## Сильные стороны
1. **Модульная структура**: ключевые подсистемы вынесены в `pkg/*`.
2. **Безопасные дефолты**: ограничение `-n` в диапазоне `1..32` и fallback логика при auto-turn.
3. **Операционные возможности**: есть route scripts и генерация v2ray-конфигов.

## Обнаруженные риски
1. **Ограниченная наблюдаемость**
   - Нет единого structured logging формата с полями stream-id/turn-id.
   - Сложнее диагностировать failover под нагрузкой.

2. **Пограничные кейсы при ротации**
   - При агрессивной ротации возможны краткие окна деградации.
   - Нужен backoff-планировщик с метрикой health-score узлов TURN.

3. **Документация и UX флагов**
   - Ранее флаги были описаны неравномерно.
   - Отсутствовала компактная таблица параметров и профилей запуска.

## Рекомендации
1. Добавить telemetry hooks (счётчики reconnect/failover, latency histogram).
2. Ввести режим `--dry-run` для проверки discovery/probe без поднятия туннеля.
3. Добавить smoke-тесты CLI (валидация обязательных флагов, генерация JSON).
4. Опционально вынести profile presets в отдельный конфиг-файл.

## Что улучшено в этом изменении
- README переработан в более читаемую структуру: разделы, таблицы флагов, профили, troubleshooting.
- Добавлены warnings и best practices для эксплуатации.
